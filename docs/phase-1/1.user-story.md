### Go Backend 實作提示：正規化引擎 (Rule Engine)

針對修改後的 US-3.1，在 Go 裡面這會是一個很有趣的小模組。建議可以設計成類似這樣的結構：

```go
// 示意結構 code snippet
type Rule struct {
    ID              int
    Pattern         string // e.g., "7-11", "^(Uber|UBER).*"
    MatchType       string // "CONTAINS", "REGEX"
    TargetMerchantID int
}

// 核心處理邏輯
func (s *NormalizationService) ResolveMerchant(rawName string, rules []Rule) (int, error) {
    for _, rule := range rules {
        matched := false
        switch rule.MatchType {
        case "CONTAINS":
            matched = strings.Contains(rawName, rule.Pattern)
        case "REGEX":
            matched, _ = regexp.MatchString(rule.Pattern, rawName)
        }
        
        if matched {
            return rule.TargetMerchantID, nil
        }
    }
    return 0, fmt.Errorf("no match found")
}

```

這個邏輯的優點是：**資料庫與邏輯解耦**。你隨時可以在前端介面增加一條 SQL 資料，後端的 Go 排程下次跑的時候就會自動套用新規則，把過去幾天「漏網」的發票重新歸類（Re-processing logic）。
