# APP/Web 功能對等驗證計畫

**日期:** 2026-03-03
**目標:** 以 Web 為基準，驗證 APP 的功能行為是否一致
**方法:** 逐頁面交替驗證（Web 先行 → APP 對照）
**工具:** Maestro MCP (APP on Emulator) + Chrome DevTools / Playwright MCP (Web)
**環境:** 同一後端 API，資料共享

---

## 執行方式

每個測試項目依照以下順序：

1. **Web 先行** — 用 Chrome DevTools/Playwright 操作頁面，記錄資料與行為
2. **APP 對照** — 用 Maestro 在 Emulator 操作相同頁面，比對結果
3. **記錄差異** — 不一致之處記錄為 issue

---

## 測試矩陣

### Tier 1: 核心流程（必測）

| # | 頁面 | 驗證項目 | Web 操作 | APP 對照 |
|---|------|---------|---------|---------|

| 6 | Invoices | 發票列表、狀態篩選、批次操作 | 篩選/選取/變更狀態 | 應一致 |

## 各頁面詳細驗證清單

### 1. Login

**Web:**
- 顯示 email 輸入框
- 送出後顯示確認訊息
- Dev mode: 直接取得 token

**APP 對照:**
- 顯示 email 輸入框
- 送出流程行為一致
- Dev mode 行為一致

### 2. Dashboard

**Web:**
- 淨資產摘要（按幣別）
- 月支出/收入統計卡片
- 待處理發票數量
- 最近交易列表

**APP 對照:**
- 月支出/收入統計（數字應一致）
- 最近交易列表（順序、金額、商家應一致）
- 注意：APP 無淨資產摘要、無趨勢圖表（預期差異）

### 3. Accounts 列表

**Web:**
- 按幣別的資產/負債摘要
- 按類型分組（CASH, BANK, CRYPTO, CREDIT）
- 帳戶卡片含餘額

**APP 對照:**
- 帳戶卡片顯示
- 餘額數字應一致
- 帳戶數量應一致

### 4. Account 詳情

**Web:**
- 帳戶基本資訊（類型、幣別、銀行、結帳日等）
- 交易歷史列表

**APP 對照:**
- 帳戶資訊應一致
- 交易歷史應一致

### 5. 建立交易

**Web:**
- 表單欄位：金額、類型、帳戶、分類、備註、日期
- 送出後交易出現在列表中

**APP 對照:**
- 表單欄位應相同
- 送出後交易應出現在 Dashboard 及帳戶詳情
- 在 Web 刷新後也應看到 APP 建立的交易

### 6. Invoices

**Web:**
- 發票列表（含搜尋、狀態篩選）
- 展開可看明細
- 批次選取與狀態變更
- 匯入按鈕

**APP 對照:**
- 發票列表顯示
- 狀態篩選功能
- 批次選取與狀態變更
- 發票數量與內容應一致

### 7. Merchants

**Web:**
- 商家列表（表格形式含名稱、預設分類、預設帳戶）
- 新增/編輯/刪除

**APP 對照:**
- 商家列表（可能為卡片或列表形式）
- CRUD 操作應生效且兩端同步

### 8. Categories

**Web:**
- 支出/收入分類各一欄
- 樹狀結構（父子分類）
- 新增/編輯/刪除

**APP 對照:**
- 分類結構應一致
- 父子關係應正確顯示
- CRUD 操作應同步

### 9. Rules

**Web:**
- 規則列表（依優先順序排列）
- 顯示 pattern → merchant 對應
- Match type badge

**APP 對照:**
- 規則列表與排序應一致
- CRUD 操作應同步

### 10. Settings

**Web:**
- Email 顯示
- 發票綁定狀態（已綁/未綁）
- 信用卡自動扣款設定

**APP 對照:**
- Email 應一致
- 綁定狀態應一致
- 注意：APP 可能無自動扣款設定（預期差異需確認）

### 11. Shared Ledgers 列表

**Web:**
- 帳本卡片（名稱、合作夥伴、幣別）
- 新增帳本表單

**APP 對照:**
- 帳本列表應一致
- 新增功能應同步

### 12. Shared Ledger 詳情

**Web:**
- 統計摘要（總額、我的份額、對方份額、應收）
- 支出列表（含分帳明細）

**APP 對照:**
- 統計數字應一致
- 支出列表應一致
- 注意：APP 無 Sheet 綁定、Alias 設定（Web-only）

### 13. 新增共帳支出

**Web:**
- 日期、類別、描述、金額
- 付款人選擇、分帳方式（4種）

**APP 對照:**
- 表單欄位應相同
- 分帳方式應一致
- 送出後兩端應同步顯示

### 14. Receivables

**Web:**
- 應收應付列表
- 結算按鈕（單筆/全部）

**APP 對照:**
- 列表應一致
- 結算操作應同步
